Étapes du projet en équipe

Jour 1 - Planifier son projet

  • Lire le cahier des charges
  • Analyser les besoins (ensemble)
  • Distinguer les différents composants du projet (ensemble)
  • Découper le projet en différentes tâches (ensemble)
  • Estimer le temps de chaque tâche
  • Répartir les tâches
  • Créer le Modèle Conceptuel de Données (MCD) avec DBDesigner ou MySQLWorkbench ou autre
  • Faire le zoning (maquettes, mockups) des pages (déjà fait, pour gagner du temps)
  • Organiser son code (créer fichiers et répertoires dès le début)
  • Utiliser les outils nécessaires au travail en équipe (Git, partage de fichiers, etc.)

Avant de coder

  • Avoir fait toute la partie "Planifier son projet" notée au dessus
  • Avoir créé les tables du projet dans la DB d'un membre de l'équipe sur wf3.progweb.fr
  • Avoir créé le repo Git
  • Avoir créé le fichier .gitignore (ignorer nbproject, config.php, vendor, etc.)
  • Avoir créé tous les dossiers et les fichiers vides
  • Avoir déposé les répertoires et fichiers sur la branche master sur le repo "origin"
  • Avoir activé sur 1 SourceTree le "GitFlow"
  • Avoir fait un "push" de la branche "develop" sur origin
  • Avoir récupéré la branche origin/develop sur chaque SourceTree
  • Avoir activé "GitFlow" sur tous les SourceTree

Jour 2 - Commencer le code

  • Commencer par la partie backend (sans habillage, juste les fonctionnalités)
  • Utiliser l'API REST omdbapi avec la fonction file_get_contents de PHP ou en Ajax
  • En option, utiliser l'API REST avec la librairie cURL de PHP
  • Décoder le contenu JSON ou XML renvoyé par l'API
  • Finir la partie backend :)

Jour 3 - Commencer le frontend

  • Commencer la partie frontend
  • Continuer la partie backend
  • Finir la partie backend avant 12h !!!!
  • Utiliser les jointures SQL (en front normalement)
  • Utiliser les sous-requêtes SQL (si besoin)
  • Bien gérer les priorités (fonctionnalités avant la cosmétique)
  • Utiliser les expressions régulières (pour la recherche)
